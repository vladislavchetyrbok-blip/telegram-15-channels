# Zodiac Visual Asset Import Checklist

After you have manually generated the visual assets for the 13 Zodiac Telegram channels using the prompts from `docs/ZODIAC_IMAGE_PROMPTS_COPYPASTE.md`, follow this checklist to safely import and validate them into the system.

## 1. Allowed Formats & Sizes
- **Format:** `.png` (Preferred for transparency/crispness on avatars) or `.jpg` (Acceptable for covers/placeholders to save space).
- **Aspect Ratio:** 
  - Avatars: 1:1 (Square, keep subject perfectly centered for circular crop)
  - Covers: 16:9 (Wide)
  - Daily Placeholders: 4:5 (Vertical)
- **Size Limit:** Keep individual files under 2MB to ensure fast loading and compliance with Telegram upload limits during the publishing phase.
- **Rule:** No raw text or typography generated by AI inside the images (unless carefully designed). Let Telegram overlay the text.

## 2. Save Locations & Naming
Place the images manually into the corresponding folders in `public/assets/zodiac/`.
File names must be **all lowercase** with **no spaces or underscores** (use hyphens).

### Avatars
Save to: `public/assets/zodiac/avatars/avatar-[channel-id].png`
*Example: `public/assets/zodiac/avatars/avatar-aries.png`*

### Covers (Optional but Recommended for Premium)
Save to: `public/assets/zodiac/covers/cover-[channel-id].jpg`
*Example: `public/assets/zodiac/covers/cover-taurus.jpg`*

### Daily Placeholders
Save to: `public/assets/zodiac/placeholders/placeholder-[channel-id].jpg`
*(Note: General channel uses `placeholder-general.jpg`)*

## 3. Validation
Once all files are moved into the folder structure, run the local validator script:

```bash
npm run zodiac:validate-assets
```

- This script will check for the existence of all expected files and strict naming compliance.
- It will output exactly which files are missing as warnings.

## 4. UI Dashboard Inspection
Finally, start your local development server:
```bash
npm run dev
```
Navigate to the **Settings > Visuals** tab (`/settings`). 
Verify that the read-only dashboard properly reflects your setup and no longer shows "missing" for the uploaded assets (assuming you updated the tracker or UI logic to read the file system, or simply rely on the CLI output as the source of truth).

*You are now ready to proceed with real Telegram channel creation and connections.*
